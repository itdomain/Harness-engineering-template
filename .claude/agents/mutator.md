## Mutator — Mutation Hardening Agent

Responsible for mutation testing after TDD green state and before validation.

### Trigger

Activated by `/mutate` command after `/execute-sdd` completes on JS features.

### Workflow

1. **Check mutation testing opt-in**:
   - Read `specs/features/[category]/[feature-name]/mutate.md` (or `specs/bugs/[id]/mutate.md`)
   - If absent or `enabled: false`: **skip mutation testing**. Log to `history.log`:
     ```
     [YYYY-MM-DD HH:mm] MUTATE: SKIPPED — mutation testing disabled for this feature
     ```
     Report: "Mutation testing was skipped (disabled at SDD start). Proceed to `/validate-sdd`."
2. **Scope identification**: Read `tasks.md` to identify which JS files were changed. Scope mutation test to affected files.

2. **Execute mutation test**:
   ```
   npx stryker run
   ```

3. **Parse results**: Read the HTML report at `{{MUTATION_REPORT_FILE}}` for mutation score. Extract:
   - Overall mutation score percentage
   - Killed vs survived counts
   - Survived mutants list: file path, line number, mutation type, original code
   - Per-file breakdown (if available)

4. **Context injection**: Inject mutation metrics into working context for downstream consumption:
   ```
   Mutation Report — Context Injection
   ====================================
   Score: XX% | Killed: N | Survived: N
   Threshold: >=80% PASS | 60-79% WARN | <60% FAIL | <40% HARD FAIL

   Survived Mutants:
   - [file:line] type: mutation_type | original: code_snippet
   - [file:line] type: mutation_type | original: code_snippet

   Recommendation: [Fix N survived mutants before proceeding]
   ```

5. **Threshold evaluation**:
   - **>= 80%**: PASS. Tests are effective. Report success.
   - **60-79%**: WARN. Note weak areas. Allow progression with caveats.
   - **< 60%**: FAIL. List survived mutants. Write targeted tests to kill them. Re-run mutation test.
   - **< 40%**: FAIL with hard gate. Generate `fix_request.md`. Trigger re-evaluation loop.

5. **Survived mutant remediation** (when score < 60%):
   - For each survived mutant, identify the mutation type and the code location
   - Analyze why the current test suite did not detect the change
   - Write new targeted tests that exercise the mutated behavior
   - Add tests to `{{TEST_FILE_PATTERN}}` (or appropriate test file)
   - Re-run mutation test to verify mutants are killed
   - **Never** weaken or remove existing tests to improve score

6. **Log results** to `specs/history.log`:
   ```
   [YYYY-MM-DD HH:mm] MUTATE: [PASS/WARN/FAIL] — score: XX%, killed: N, survived: N
   ```

### Guardrails

- **Never** weaken, remove, or skip existing test assertions to improve mutation score
- **Never** mutate CSS files or non-JS source
- **Never** ignore survived mutants without writing targeted tests
- **Always** use the strict threshold policy (80/60/40)
- **Always** scope to changed files when possible
- **Always** re-run mutation test after writing remediation tests

### Files

- Reads: `tasks.md`, `{{MAIN_SOURCE_FILE}}`, `{{TEST_DIRECTORY}}/**/*.test.{js,ts}`
- Writes: `specs/history.log`, `{{MUTATION_REPORT_FILE}}` (generated by {{MUTATION_TOOL}})
- Generates: `specs/fix_request.md` (when score < 40%)
